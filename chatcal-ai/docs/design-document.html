🌟 ChatCal.ai - Technical Design Document

Executive Summary

ChatCal.ai is an AI-powered appointment booking assistant that integrates with Google Calendar. Built with Python, LlamaIndex, and Anthropic's Claude, it provides a friendly, conversational interface for scheduling meetings. The application is fully containerized for easy deployment across any platform.

API Design

RESTful Endpoints

POST /api/chat
Send message to chatbot
Body: { "message": "string", "session_id": "string" }
GET /api/auth/login
Initiate Google OAuth flow
Returns: Redirect to Google authorization
GET /api/auth/callback
Handle OAuth callback
Query: ?code=auth_code&state=session_state
GET /api/sessions/{session_id}
Retrieve conversation history
Returns: { "messages": [], "created_at": "timestamp" }
GET /api/health
Health check endpoint
Returns: { "status": "healthy", "version": "0.1.0" }

Conversation Flow

Typical Booking Flow

  1. User: "I need to schedule a meeting with Sarah next week"
  2. Bot: "I'd be delighted to help! 🌟 What day works best for you?"
  3. User: "Tuesday afternoon would be great"
  4. Bot: "Perfect! I see you have availability at 2pm, 3pm, and 4pm on Tuesday. Which time suits you best?"
  5. User: "Let's do 3pm"
  6. Bot: "How long should I schedule this meeting for? 30 minutes? An hour?"
  7. User: "One hour please"
  8. Bot: "Fantastic! 🎉 Your meeting with Sarah is all set for Tuesday at 3pm for one hour. You're crushing this scheduling game!"

Data Flow & Processing

Request Processing Pipeline

1. User Message → FastAPI Endpoint
2. Session Validation (Redis)
3. Context Retrieval (Previous Messages)
4. LlamaIndex Agent Processing
   ├── Intent Recognition
   ├── Entity Extraction (Time, Person, Duration)
   └── Calendar Tool Invocation
5. Google Calendar API Call
6. Response Generation (Claude)
7. Personality Filter Application
8. Response to User

Deployment Architecture

☁️ Cloud Deployment Options

Security Considerations

🔒 Security Measures

  • OAuth2 for Google Calendar authentication
  • API keys stored in environment variables
  • HTTPS enforcement in production
  • Session tokens with expiration
  • Rate limiting on API endpoints
  • Input validation and sanitization
  • No storage of sensitive calendar data

Performance Considerations

  • Response Time: Target < 2 seconds for chat responses
  • Concurrent Users: Redis sessions support 1000+ concurrent users
  • API Rate Limits: Google Calendar API: 1,000,000 queries/day
  • Caching: Calendar availability cached for 5 minutes
  • Scaling: Horizontal scaling via container orchestration
